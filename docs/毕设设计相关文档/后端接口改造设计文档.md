## 改造一：报告初审+终审双级审核机制

### 1.1 问题描述

当前 `ReportAuditServiceImpl` 只实现了单级审核（`pending_review → audited`），任务书要求**初审（检验医师）+ 终审（审核医师/主管）**两级审核。

### 1.2 现有状态流转

```
draft → pending_review → audited → reported → printed
                ↓
              draft（驳回）
```

### 1.3 目标状态流转

```
draft → pending_review → first_audited → final_audited → reported → printed
                ↓              ↓
              draft          pending_review（终审驳回）
```

| 状态 | 中文 | 说明 |
|------|------|------|
| draft | 草稿 | 初始状态/驳回后回退 |
| pending_review | 待初审 | 提交审核后 |
| first_audited | 已初审 | 初审通过，等待终审 |
| final_audited | 已终审 | 终审通过，可发布 |
| reported | 已发布 | 发布后 |
| printed | 已打印 | 打印后 |
| cancelled | 已取消 | 取消 |

#### (2) 新增状态校验方法

```java
private void validateStatusTransition(String currentStatus, String targetStatus) {
    SpecimenStatusEnum current = SpecimenStatusEnum.fromCode(currentStatus);
    SpecimenStatusEnum target = SpecimenStatusEnum.fromCode(targetStatus);
    Set<SpecimenStatusEnum> allowed = ALLOWED_TRANSITIONS.get(current);
    if (allowed == null || !allowed.contains(target)) {
        throw new BusinessException(
            String.format("标本状态不允许从[%s]变更为[%s]", current.getDesc(), target.getDesc()));
    }
}
```

## 改造三：Gateway Token黑名单校验

### 3.1 问题描述

Auth 服务登出时将 AccessToken 加入 Redis 黑名单（key: `blacklist:token:{完整token}`），但 Gateway 的 `JwtAuthenticationFilter` 没有检查黑名单，导致登出后 Token 在过期前仍然有效。

**现有 Auth 登出逻辑**（`AuthServiceImpl.java` L178-198）：

```java
// 现有代码 - 已实现黑名单写入
stringRedisTemplate.opsForValue().set(
    ACCESS_TOKEN_BLACKLIST_PREFIX + token,  // key = "blacklist:token:" + 完整token
    "1",
    expiration,                             // 过期时间 = accessTokenExpiration
    TimeUnit.MILLISECONDS
);
```

**现有 Redis 常量**（`RedisConstants.java`）：

```java
public static final String BLACKLIST_PREFIX = "blacklist:";
// 实际黑名单key = "blacklist:token:" + 完整token字符串
```

### 3.2 JwtAuthenticationFilter 变更

**文件**：`lis-gateway/.../filter/JwtAuthenticationFilter.java`

注入 `StringRedisTemplate`，在 Token 解析成功后增加黑名单检查：

```java
private final StringRedisTemplate redisTemplate;

// 构造函数增加 redisTemplate 参数
public JwtAuthenticationFilter(JwtProperties jwtProperties,
                                GatewayProperties gatewayProperties,
                                JwtUtil jwtUtil,
                                StringRedisTemplate redisTemplate) {
    this.jwtProperties = jwtProperties;
    this.gatewayProperties = gatewayProperties;
    this.jwtUtil = jwtUtil;
    this.redisTemplate = redisTemplate;
}

@Override
public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {
    // ... 原有白名单和Token提取逻辑不变 ...

    try {
        Claims claims = jwtUtil.parseToken(token, jwtProperties.getSecret());

        if (jwtUtil.isTokenExpired(claims)) {
            return unauthorized(exchange, "令牌已过期");
        }

        // ===== 新增：检查Token黑名单 =====
        // Auth服务登出时写入的key = "blacklist:token:" + 完整token
        String blacklistKey = "blacklist:token:" + token;
        Boolean isBlacklisted = redisTemplate.hasKey(blacklistKey);
        if (Boolean.TRUE.equals(isBlacklisted)) {
            return unauthorized(exchange, "令牌已失效");
        }
        // ===== 新增结束 =====

        Long userId = jwtUtil.getUserId(claims);
        String username = jwtUtil.getUsername(claims);

        // ... 后续逻辑不变 ...
    } catch (...) {
        // ... 不变 ...
    }
}
```

> **注意**：Auth 服务已使用 `blacklist:token:` + 完整token 作为黑名单 key，Gateway 只需用相同规则检查即可。无需修改 Auth 服务的登出逻辑，也无需在 JwtTokenProvider 中添加 jti。

### 3.3 JwtTokenProvider 无需变更

**文件**：`lis-auth/.../util/JwtTokenProvider.java`

## 改造八：模拟HIS检验申请功能（HL7 ORM消息闭环）

### 8.1 问题描述

任务书要求"模拟HIS与实验室系统交互成功率≥98%"，当前系统存在以下关键缺陷：

1. **Hl7MessageBuilder 缺少 `buildOrmO01()` 方法**：构建 ORM 消息只能通过 `buildCustomMessage()` 字符串拼接，无法保证 HL7 协议合规性
2. **入站消息处理断裂**：`DefaultMllpMessageHandler` 只解析+存储消息，不调用业务逻辑；`HisIntegrationService.processOrmMessage()` 只打印日志
3. **HL7 消息与业务系统未打通**：ORM 消息接收后不会自动创建标本登记或检验申请
4. **缺少值映射转换层**：HL7 文本字段（如科室名称、标本类型名称）无法自动映射为业务系统 ID

### 8.2 目标业务流程

### 8.5 Hl7MessageBuilder.buildOrmO01() 实现

使用 HAPI 对象模型构建标准 ORM^O01 消息：

```java
public String buildOrmO01(HisLabOrderDTO order) {
    try {
        ORM_O01 ormMsg = new ORM_O01();
        ormMsg.initQuickstart("ORM", "O01", "P");

        // MSH 段
        MSH msh = ormMsg.getMSH();
        msh.getSendingApplication().getNamespaceID().setValue("HIS");
        msh.getSendingFacility().getNamespaceID().setValue("HIS_HOSP");
        msh.getReceivingApplication().getNamespaceID().setValue("LIS");
        msh.getReceivingFacility().getNamespaceID().setValue("LIS_LAB");
        msh.getDateTimeOfMessage().getTime().setValue(
            LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss")));
        msh.getMessageControlID().setValue(generateMessageControlId());

        // PID 段
        PID pid = ormMsg.getPATIENT().getPID();
        pid.getPatientID().getIDNumber().setValue(order.getPatientId());
        XPN patientName = pid.getPatientName(0);
        patientName.getFamilyName().getSurname().setValue(order.getPatientName());
        pid.getDateTimeOfBirth().getTime().setValue(order.getPatientAge());
        pid.getAdministrativeSex().setValue(order.getPatientGender());
        pid.getPatientAddress(0).getValue(); // 地址

        // PV1 段
        PV1 pv1 = ormMsg.getPATIENT().getPATIENT_VISIT().getPV1();
        pv1.getPatientClass().setValue(order.getVisitType());
        pv1.getAssignedPatientLocation().getPointOfCare().setValue(order.getWardName());
        pv1.getAssignedPatientLocation().getFacility().getNamespaceID().setValue(order.getDeptName());
        pv1.getVisitNumber().getIDNumber().setValue(order.getVisitNo());
        pv1.getAttendingPhysician(0).getFamilyName().getSurname().setValue(order.getDoctorName());

        // ORDER Detail - 每个 orderItem 对应一个 ORC+OBR 对
        for (int i = 0; i < order.getOrderItems().size(); i++) {
            HisLabOrderItemDTO item = order.getOrderItems().get(i);
            ORM_O01_ORDER orderDetail = ormMsg.getORDER(i);

            // ORC 段
            ORC orc = orderDetail.getORC();
            orc.getOrderControl().setValue("NW");
            orc.getPlacerOrderNumber().getEntityIdentifier().setValue("HIS" + System.currentTimeMillis());
            orc.getOrderingProvider(0).getFamilyName().getSurname().setValue(order.getDoctorName());

            // OBR 段
            OBR obr = orderDetail.getORDER_DETAIL().getOBR();
            obr.getFillerOrderNumber().getEntityIdentifier().setValue("LIS" + System.currentTimeMillis());
            obr.getUniversalServiceIdentifier().getIdentifier().setValue(item.getItemCode());
            obr.getUniversalServiceIdentifier().getText().setValue(item.getItemName());
            obr.getPriority().setValue(item.getPriority());
            obr.getSpecimenSource().getSpecimen().getText().setValue(order.getSpecimenTypeName());
            obr.getClinicalInformation().setValue(order.getClinicalDiagnosis());
        }

        return new PipeParser().encode(ormMsg);
    } catch (Exception e) {
        throw new BusinessException("构建ORM消息失败: " + e.getMessage());
    }
}
```

### 8.9 HisIntegrationService.processOrmMessage() 业务实现

```java
private void processOrmMessage(Hl7MessageDO messageDO, Map<String, Object> parsedMessage) {
    log.info("处理ORM检验申请消息, messageControlId={}", messageDO.getMessageControlId());

    try {
        // 1. 转换为标本登记DTO
        SpecimenRegisterDTO specimenDTO = hl7ToSpecimenConverter.convert(parsedMessage);
        
        // 2. 通过Feign调用标本登记
        Result<SpecimenVO> specimenResult = specimenFeignClient.registerSpecimen(specimenDTO);
        if (specimenResult.getCode() != 200) {
            log.error("标本登记失败: {}", specimenResult.getMessage());
            messageDO.setProcessStatus("failed");
            messageDO.setErrorMsg("标本登记失败: " + specimenResult.getMessage());
            return;
        }

        // 3. 转换为检验申请DTO
        ReportApplyDTO reportDTO = hl7ToReportConverter.convert(parsedMessage, specimenResult.getData());
        
        // 4. 通过Feign调用检验申请
        Result<ReportVO> reportResult = reportFeignClient.createReportApply(reportDTO);
        if (reportResult.getCode() != 200) {
            log.error("检验申请创建失败: {}", reportResult.getMessage());
            messageDO.setProcessStatus("failed");
            messageDO.setErrorMsg("检验申请创建失败: " + reportResult.getMessage());
            return;
        }

        messageDO.setProcessStatus("success");
        messageDO.setSpecimenNo(specimenResult.getData().getSpecimenNo());
        log.info("ORM消息处理成功, specimenNo={}, reportNo={}",
            specimenResult.getData().getSpecimenNo(), reportResult.getData().getReportNo());
    } catch (Exception e) {
        log.error("ORM消息处理异常", e);
        messageDO.setProcessStatus("failed");
        messageDO.setErrorMsg("处理异常: " + e.getMessage());
    }
}
```

### 8.10 Hl7ToSpecimenConverter 值映射转换器

**核心问题**：HL7 消息中的科室、标本类型、检验项目都是文本，业务系统需要 ID。

**解决方案**：通过 Feign 调用 lis-specimen 的查询接口，按名称模糊匹配获取 ID。

```java
@Component
public class Hl7ToSpecimenConverter {

    @Autowired
    private SpecimenFeignClient specimenFeignClient;

    public SpecimenRegisterDTO convert(Map<String, Object> parsedMessage) {
        Map<String, Object> patient = (Map<String, Object>) parsedMessage.get("patient");
        Map<String, Object> visit = (Map<String, Object>) parsedMessage.get("visit");
        List<Map<String, Object>> orders = (List<Map<String, Object>>) parsedMessage.get("orders");

        SpecimenRegisterDTO dto = new SpecimenRegisterDTO();

        // 患者信息
        dto.setPatientName(getStr(patient, "patientName"));
        dto.setPatientGender(convertGender(getStr(patient, "gender")));
        dto.setPatientAge(getStr(patient, "birthDate"));
        dto.setPatientIdNo(getStr(patient, "idNo"));
        dto.setPatientPhone(getStr(patient, "phone"));

        // 科室信息 - 通过名称查找ID
        String deptName = getStr(visit, "department");
        dto.setDeptName(deptName);
        dto.setDeptId(resolveDeptId(deptName));

        dto.setWardName(getStr(visit, "ward"));
        dto.setBedNo(getStr(visit, "bed"));
        dto.setDoctorName(getStr(visit, "attendingDoctor"));

        // 标本类型 - 通过名称查找ID
        String specimenTypeName = orders.isEmpty() ? null :
            getStr(orders.get(0), "specimenType");
        dto.setSpecimenTypeName(specimenTypeName);
        dto.setSpecimenTypeId(resolveSpecimenTypeId(specimenTypeName));

        // 检验项目 - 通过名称查找ID
        List<Long> testItemIds = orders.stream()
            .map(order -> resolveTestItemId(getStr(order, "orderItemName")))
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
        dto.setTestItemIds(testItemIds);

        // 其他信息
        dto.setClinicalDiagnosis(orders.isEmpty() ? null :
            getStr(orders.get(0), "clinicalInfo"));
        dto.setIsStat(convertPriority(orders.isEmpty() ? null :
            getStr(orders.get(0), "priority")));
        dto.setCollectTime(LocalDateTime.now());

        return dto;
    }

    private Long resolveDeptId(String deptName) {
        if (StringUtils.hasText(deptName)) {

            } catch (Exception e) {
                log.warn("通过名称[{}]查找科室ID失败", deptName);
            }
        }
        return 1L; // 默认科室ID
    }
```

### 8.14 错误处理机制

| 异常场景 | 处理策略 |
|---------|---------|
| ORM 消息格式错误 | 返回 AE（Application Error）ACK，记录错误信息到 `hl7_message.error_msg` |
| 科室/标本类型名称无法匹配 | 使用默认值（科室ID=1），在 `remark` 中标注"HL7自动匹配，需人工确认" |

| Feign 调用检验申请失败 | 消息状态设为 `failed`，记录错误信息，支持重处理 |
| MLLP 连接超时 | `HisMessageClient` 抛出异常，消息状态设为 `failed`，支持重试 |
| ACK 返回 AE/AR | 消息状态设为 `failed`，记录 ACK 错误信息 |

## 改造九：系统日志功能（五类日志体系）

### 9.1 问题描述

任务书要求"关键操作留存日志，支持溯源"。当前系统日志功能严重缺失：

#### 9.6.2 OperLogAspect 切面

```java
@Aspect
@Component
public class OperLogAspect {

    @Autowired
    private LogFeignClient logFeignClient;

    @AfterReturning(pointcut = "@annotation(operLog)", returning = "result")
    public void doAfterReturning(JoinPoint joinPoint, OperLog operLog, Object result) {
        handleLog(joinPoint, operLog, null, result);
    }

    @AfterThrowing(pointcut = "@annotation(operLog)", throwing = "e")
    public void doAfterThrowing(JoinPoint joinPoint, OperLog operLog, Exception e) {
        handleLog(joinPoint, operLog, e, null);
    }

    private void handleLog(JoinPoint joinPoint, OperLog operLog, Exception e, Object result) {
        OperLogMessage message = new OperLogMessage();
        message.setTitle(operLog.title());
        message.setBusinessType(operLog.businessType());
        message.setMethod(joinPoint.getSignature().getDeclaringTypeName() + "." + joinPoint.getSignature().getName());
        message.setRequestMethod(RequestContextHolder.getRequestAttributes()...getMethod());
        message.setOperatorType(1); // 后台用户

        // 从请求头获取用户信息
        ServletRequestAttributes attrs = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (attrs != null) {
            HttpServletRequest request = attrs.getRequest();
            message.setOperName(request.getHeader("X-User-Name"));
            message.setOperUrl(request.getRequestURI());
            message.setOperIp(IpUtils.getIpAddr(request));
            if (operLog.isSaveRequestData()) {
                message.setOperParam(getMethodParams(joinPoint));
            }
            if (operLog.isSaveResponseData() && result != null) {
                message.setJsonResult(JsonUtils.toJson(result));
            }
        }

        if (e != null) {
            message.setStatus(1);
            message.setErrorMsg(e.getMessage());
        } else {
            message.setStatus(0);
        }

        message.setOperTime(LocalDateTime.now());
        message.setCostTime(calculateCostTime());

        // 异步保存
        logFeignClient.saveOperLog(message);
    }
}
```

### 9.8 系统日志（错误日志）接口

#### 9.8.1 GlobalErrorLogAspect 切面

拦截所有 Controller 层的未捕获异常，自动记录错误日志：

```java
@Aspect
@Component
public class GlobalErrorLogAspect {

    @Autowired
    private LogFeignClient logFeignClient;

    @AfterThrowing(pointcut = "execution(* com.lis..controller..*.*(..))", throwing = "e")
    public void logError(JoinPoint joinPoint, Exception e) {
        ErrorLogMessage message = new ErrorLogMessage();
        message.setErrorTime(LocalDateTime.now());
        message.setErrorLevel(e instanceof BusinessException ? "WARN" : "ERROR");
        message.setErrorClass(e.getClass().getName());
        message.setErrorMethod(joinPoint.getSignature().getDeclaringTypeName() + "." + joinPoint.getSignature().getName());
        message.setErrorMessage(e.getMessage());
        message.setErrorStack(ExceptionUtils.getStackTrace(e));

        // 从请求上下文获取用户信息和请求信息
        ServletRequestAttributes attrs = (ServletRequestAttributes) RequestContextHolder.getRequestAttributes();
        if (attrs != null) {
            HttpServletRequest request = attrs.getRequest();
            message.setRequestUrl(request.getRequestURI());
            message.setRequestMethod(request.getMethod());
            message.setUserId(getUserIdFromHeader(request));
            message.setUserName(request.getHeader("X-User-Name"));
            message.setUserIp(IpUtils.getIpAddr(request));
        }

        message.setServerNode(getServerNode());
        message.setThreadName(Thread.currentThread().getName());

        // 异步保存
        logFeignClient.saveErrorLog(message);
    }
}
```

### 9.11 日志查询通用参数

所有日志查询接口支持以下通用筛选参数：

| 参数 | 类型 | 说明 |

**审计日志**：auditType（审计类型）、targetType（对象类型）、operatorName（操作人）、action（操作动作）

**接口日志**：direction（方向）、messageType（消息类型）、processStatus（处理状态）、patientName（患者姓名）
