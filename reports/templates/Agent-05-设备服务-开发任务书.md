## 二、任务背景

### 2.1 业务背景

在LIS（实验室信息管理系统）中，设备管理是核心基础模块之一。检验仪器设备是实验室开展检验工作的物质基础，设备的正常运行状态直接关系到检验结果的准确性和可靠性。设备服务Agent负责管理各类检验仪器（如血细胞分析仪、生化分析仪、尿液分析仪、免疫分析仪等）的全生命周期管理，包括设备台账管理、状态监控、联机通讯、数据采集与解析、校准记录管理以及维护保养记录管理等功能。

### 2.2 技术背景

设备管理微服务采用Spring Boot框架构建，作为独立微服务部署运行。设备联机通讯涉及RS232串口通讯和TCP网络通讯两种方式，需要处理不同厂商仪器的数据协议（主要为ASTM协议）。服务需要具备高可用性、良好的异常处理能力和完善的日志记录机制。

### 2.3 核心目标

1. 建立完整的设备台账管理体系，实现设备信息的规范化管理
2. 实现设备与LIS系统的联机通讯，自动采集检验数据
3. 提供设备状态实时监控与告警能力
4. 建立设备校准与维护保养的完整记录链
5. 提供设备使用统计分析，为设备管理决策提供数据支撑

## 三、任务清单

### 任务1：设备CRUD

#### 详细描述

实现设备基本信息的增删改查功能，包括设备列表分页查询、设备新增、设备信息修改、设备删除等操作。设备信息包括设备编号、设备名称、设备型号、生产厂家、序列号、安装日期、设备位置、通讯参数、设备类型等。

#### 输入

- **新增设备**：设备编号（自动生成或手动输入）、设备名称、设备型号、生产厂家、序列号、安装日期、设备位置、设备类型、通讯方式（RS232/TCP）、通讯参数（波特率、数据位、停止位、校验位/IP地址、端口号）
- **修改设备**：设备ID及需要修改的字段信息
- **删除设备**：设备ID（逻辑删除）
- **查询设备**：查询条件（设备名称、设备型号、设备状态、设备类型）、分页参数（页码、每页条数）

#### 输出

- **新增结果**：操作成功/失败状态、新增设备的完整信息
- **修改结果**：操作成功/失败状态、修改后的设备完整信息
- **删除结果**：操作成功/失败状态
- **查询结果**：设备列表（分页）、总记录数、总页数

#### 技术要点

- 使用MyBatis-Plus实现分页查询，支持多条件组合查询
- 设备编号采用"EQ+年月日+4位流水号"的格式自动生成
- 删除操作采用逻辑删除（deleted字段），保留历史数据
- 使用Spring Validation进行参数校验（@NotNull、@NotBlank、@Pattern等）
- 设备通讯参数使用JSON格式存储，支持灵活配置
- 接口采用RESTful风格设计：POST /api/equipment（新增）、PUT /api/equipment/{id}（修改）、DELETE /api/equipment/{id}（删除）、GET /api/equipment/page（分页查询）

### 任务2：设备状态管理

#### 详细描述

实现设备状态的完整生命周期管理，支持设备在以下状态之间流转：启用（正常使用）、停用（暂停使用）、维修中（正在维修）、报废（不再使用）。每次状态变更需记录变更原因、操作人、操作时间等信息。

#### 输入

- **状态变更请求**：设备ID、目标状态、变更原因、操作人ID
- **状态查询请求**：设备ID或设备编号

#### 输出

- **状态变更结果**：操作成功/失败状态、变更后的设备状态、状态变更时间
- **状态查询结果**：设备当前状态、状态变更历史记录列表

#### 技术要点

- 使用枚举类型定义设备状态（ENABLED-启用、DISABLED-停用、MAINTAINING-维修中、SCRAPPED-报废）
- 实现状态机模式，定义合法的状态流转路径（如：启用→停用、启用→维修中、维修中→启用、维修中→报废等）
- 非法状态流转需抛出业务异常并给出明确提示
- 状态变更记录写入独立的设备状态变更日志表
- 设备维修中状态时，自动停止该设备的联机通讯任务
- 设备报废后，禁止任何操作并标记为不可用

### 任务3：设备联机通讯

#### 详细描述

实现设备与LIS系统之间的数据通讯功能，支持RS232串口通讯和TCP网络通讯两种方式。通讯功能包括连接管理（建立连接、断开连接、重连机制）、心跳检测、数据接收与发送等。每个设备可配置独立的通讯参数，系统根据设备配置自动选择通讯方式。

#### 输入

- **连接请求**：设备ID、通讯方式（RS232/TCP）、通讯参数
- **断开请求**：设备ID
- **数据发送**：设备ID、发送数据内容
- **心跳检测**：设备ID、检测间隔时间

#### 输出

- **连接结果**：连接成功/失败状态、连接时间、通讯通道标识
- **断开结果**：断开成功/失败状态、断开时间
- **接收数据**：原始数据报文、接收时间、数据长度
- **发送数据**：发送结果、发送时间
- **心跳检测结果**：设备在线/离线状态、最后通讯时间

#### 技术要点

- **RS232串口通讯**：使用jssc（Java Simple Serial Connector）库实现串口操作，支持配置波特率（9600/19200/38400/115200）、数据位（7/8）、停止位（1/2）、校验位（None/Odd/Even）
- **TCP网络通讯**：使用Netty框架实现TCP服务端/客户端，支持长连接和短连接两种模式
- **连接管理**：使用连接池管理设备连接，支持最大连接数限制
- **心跳检测**：定时发送心跳包检测设备在线状态，超时未响应标记为离线并触发告警
- **断线重连**：连接断开后自动重连，采用指数退避算法（初始间隔5秒，最大间隔60秒，最大重试次数10次）
- **通讯线程管理**：每个设备使用独立的通讯线程，避免相互干扰
- **数据缓冲**：接收数据使用环形缓冲区，防止数据丢失

### 任务4：仪器数据解析

#### 详细描述

实现从检验仪器接收到的原始数据报文的解析功能，主要支持ASTM协议格式的数据解析。将解析后的数据转换为LIS系统内部标准格式，供后续业务流程使用。支持多种常见检验仪器的数据格式。

#### 输入

- **原始报文**：设备发送的原始数据字符串（ASTM格式）
- **设备信息**：设备ID、设备型号（用于确定数据格式）
- **报文类型**：查询请求/结果数据/确认应答

#### 输出

- **解析结果**：结构化的检验数据（患者信息、标本信息、检验项目、检验结果、参考范围、异常标志等）
- **解析状态**：解析成功/失败、失败原因
- **格式转换结果**：LIS系统内部标准数据格式

#### 技术要点

- **ASTM协议解析**：实现ASTM E1381/E1394协议解析器，支持帧结构解析（STX起始符、ETX终止符、CR/LF、帧校验）
- **记录类型处理**：支持H（头记录）、P（患者记录）、O（订单记录）、R（结果记录）、C（注释记录）、L（尾记录）、M（设备信息记录）等记录类型
- **数据分段**：支持多帧数据拼接（中间帧使用ETB结束，最后一帧使用ETX结束）
- **校验机制**：支持LRC（纵向冗余校验）和CRC校验
- **数据映射**：建立仪器检验项目代码与LIS系统检验项目代码的映射关系表
- **异常数据处理**：对格式异常的数据进行容错处理，记录异常日志
- **解析器扩展**：采用策略模式设计，便于扩展支持不同厂商的私有协议

### 任务5：校准记录管理

#### 详细描述

实现设备校准记录的管理功能，包括校准记录的新增和查询。校准记录记录设备每次校准的详细信息，包括校准日期、校准类型、校准项目、校准结果、校准人员等信息。

#### 输入

- **新增校准记录**：设备ID、校准日期、校准类型（日常校准/定期校准/故障后校准）、校准项目列表（项目名称、标准值、实测值、偏差、是否合格）、校准人员、校准备注
- **查询校准记录**：设备ID、时间范围、校准类型、分页参数

#### 输出

- **新增结果**：操作成功/失败状态、校准记录ID
- **查询结果**：校准记录列表（分页）、总记录数

#### 技术要点

- 校准项目数据使用JSON格式存储，支持灵活的项目数量
- 校准记录关联设备信息，支持通过设备ID或设备编号查询
- 校准记录一旦保存不可修改，仅支持新增和查询（保证数据完整性）
- 支持按时间范围、校准类型等多条件组合查询
- 校准结果自动判定（偏差在允许范围内为合格，否则为不合格）
- 校准不合格时自动触发设备状态变更为"维修中"的提醒

### 任务6：维护保养记录管理

#### 详细描述

实现设备维护保养记录的管理功能，包括维护保养记录的新增和查询。维护保养记录记录设备每次维护保养的详细信息，包括维护日期、维护类型、维护内容、维护人员、维护费用等。

#### 输入

- **新增维护记录**：设备ID、维护日期、维护类型（日常保养/定期维护/故障维修/预防性维护）、维护内容描述、更换配件信息（配件名称、配件编号、数量、费用）、维护人员、维护费用、维护结果（正常/异常）、下次维护日期
- **查询维护记录**：设备ID、时间范围、维护类型、分页参数

#### 输出

- **新增结果**：操作成功/失败状态、维护记录ID
- **查询结果**：维护记录列表（分页）、总记录数

#### 技术要点

- 维护记录关联设备信息，支持通过设备ID查询该设备的所有维护记录
- 更换配件信息使用JSON格式存储，支持多个配件记录
- 维护记录支持按维护类型分类查询和统计
- 维护完成后可自动更新设备的下次维护提醒日期
- 维护费用支持汇总统计，便于成本分析
- 维护记录支持导出功能（Excel格式）

---

### 任务7：设备使用统计

#### 详细描述

实现设备使用情况的统计分析功能，统计设备的使用频次、运行时长、故障次数、数据采集量等指标，为设备管理决策提供数据支撑。

#### 输入

- **统计请求**：设备ID或全部设备、统计时间范围（日/周/月/季度/年）、统计维度（使用次数、运行时长、故障次数、数据采集量）

#### 输出

- **统计结果**：设备使用统计数据列表，包含设备名称、统计周期、使用次数、运行时长、故障次数、数据采集量等
- **汇总数据**：设备使用率排名、故障率排名

#### 技术要点

- 使用MyBatis聚合查询进行数据统计，避免全表扫描
- 统计数据定时计算并缓存（使用Redis缓存，缓存时间1小时）
- 支持多维度交叉统计（设备+时间+科室等维度）
- 统计结果支持图表展示（折线图展示趋势、柱状图展示对比、饼图展示占比）
- 提供设备使用率预警功能（使用率过高或过低时触发告警）
- 统计数据支持按日、周、月、季度、年等不同时间粒度输出

- RESTful API设计风格
- 统一响应格式：{code, message, data}
- 接口版本号：/api/equipment

- 敏感操作需记录操作日志

- 通讯参数配置信息加密存储
- 设备连接认证（IP白名单/设备序列号验证）
- 操作权限校验（基于角色的访问控制）

## 【Git工作流规范】

> 本章节为V3.0新增，所有规则具有强制约束力。详细规范参见 `/workspace/lis-project/docs/05-开发规范/Git版本控制规范.md`。

### 分支管理

- **工作分支**：`feature/agent-05-equipment-service`
- **修复分支**：`bugfix/agent-05-{description}`
- **禁止**直接在`main`或`develop`分支上提交任何代码
- 分支由项目经理（PM）创建，你只能在分配的分支上工作

### 提交规范

- **格式**：`[Agent-05] {类型}({范围}): {简述}`
- **类型**：feat（新功能）、fix（修复）、refactor（重构）、docs（文档）、test（测试）、chore（构建/配置）
- **示例**：`[Agent-05] feat(设备服务): 新增XXX功能`
- 每个功能点至少一次提交，禁止积压大量变更后一次性提交
- 提交前必须确保代码可编译、功能可运行

### 过程文档要求

每个功能点完成后，必须同步提交过程文档：

| 文档类型 | 存放路径 | 提交时机 |
|---------|---------|---------|
| 功能开发记录 | `/workspace/lis-project/docs/process/Agent-05/{功能名称}-开发记录.md` | 每个功能点完成时 |
| 变更日志 | `/workspace/lis-project/docs/process/Agent-05/CHANGELOG.md` | 每次提交时更新 |

过程文档模板参见：`/workspace/lis-project/docs/process/开发记录模板.md`

### 阶段交付要求

完成本阶段全部任务后，必须提交以下交付物（缺一不可）：

1. ✅ 阶段开发报告（按D09《开发报告模板》格式）
2. ✅ CHANGELOG变更日志
3. ✅ 所有功能点的开发记录（`docs/process/Agent-05/`目录下）
4. ✅ 数据库变更脚本（如有数据库结构变更，含回滚脚本）
5. ✅ 通知项目经理进行合并审查

### PM汇报机制

- **每完成一个功能点**：Git commit + 过程文档 + 通知PM（附commit hash）
- **遇到阻塞问题**：立即通知PM，不得自行跳过或绕过
- **每日结束前**：提交当日进度总结到CHANGELOG
- **合并请求**：全部完成后，等待PM审查代码和文档，经PM审批后方可合并到develop

### 违规后果

| 违规行为 | 后果 |
|---------|------|
| 直接在main/develop上提交 | PM执行git reset回退 + 警告 |
| 提交信息不规范 | PM驳回提交，要求修改 |
| 过程文档缺失 | PM驳回合并请求 |
| 未经审批自行合并 | PM回退合并 + 暂停Agent |
| 虚假代码/文档 | PM回退分支 + 按红线禁令R1处理 |
