### 四、遇到的问题及解决方案

**问题1：Nacos 服务注册偶发失败**

- **现象**：在开发环境中，微服务启动时偶尔出现注册 Nacos 失败的情况，导致服务间调用不通。
- **原因分析**：Nacos 服务端默认使用嵌入式数据库（Derby），在频繁重启场景下可能出现数据不一致。
- **解决方案**：将 Nacos 数据源切换为 MySQL 持久化存储，并调整服务注册重试策略（重试间隔 3 秒，最大重试次数 5 次）。

**问题2：JWT 令牌刷新机制设计不合理**

- **现象**：初期采用固定过期时间方案，用户在操作过程中频繁被要求重新登录。
- **原因分析**：未设计合理的令牌刷新机制，Access Token 过期后直接要求用户重新认证。
- **解决方案**：引入双令牌机制，Access Token 有效期 30 分钟，Refresh Token 有效期 7 天。前端在 Access Token 过期前 5 分钟自动通过 Refresh Token 刷新，实现无感续期。

**问题3：分布式事务一致性保障困难**

- **现象**：标本登记时需要同时操作用户服务和标本服务，偶发数据不一致问题。
- **原因分析**：跨服务调用缺乏事务一致性保障。
- **解决方案**：引入 Seata 分布式事务框架，采用 AT 模式实现最终一致性。对强一致性要求的场景使用 TCC 模式。

### 四、遇到的问题及解决方案

**问题1：前端跨域请求被拦截**

- **现象**：前端开发环境调用后端 API 时出现 CORS 跨域错误。
- **原因分析**：Gateway 网关未正确配置跨域策略，导致预检请求（OPTIONS）被拒绝。
- **解决方案**：在 Gateway 中配置全局 CORS 过滤器，允许指定域名跨域访问，并正确处理预检请求。

**问题2：HL7 消息解析兼容性问题**

- **现象**：对接不同厂商的检验仪器时，HL7 消息格式存在细微差异，导致部分消息解析失败。
- **原因分析**：各厂商对 HL7 标准的实现存在差异，部分字段使用自定义分隔符或扩展编码。
- **解决方案**：设计可配置的 HL7 消息解析器，支持通过配置文件自定义字段映射规则，提高兼容性。

**问题3：统计查询性能瓶颈**

- **现象**：当统计时间范围超过 6 个月时，部分统计接口响应时间超过 5 秒。
- **原因分析**：统计查询涉及大量数据聚合计算，且每次请求都重新计算。
- **解决方案**：引入 Redis 缓存层，对统计结果进行缓存（缓存时间 10 分钟），并采用定时任务预计算常用统计指标。

**问题4：AI 模型推理延迟较高**

- **现象**：智能辅助诊断接口平均响应时间约 2.3 秒，影响用户体验。
- **原因分析**：模型推理在 CPU 环境下运行，计算资源有限。
- **解决方案**：将 AI 服务独立部署，采用异步推理模式，前端通过轮询获取结果。同时优化模型体积，采用量化压缩后的轻量级模型。

### 四、遇到的问题及解决方案

**问题1：Docker 容器间网络通信延迟**

- **现象**：容器化部署后，服务间调用延迟较本地开发环境增加约 50ms。
- **原因分析**：Docker 默认 bridge 网络模式存在 DNS 解析延迟。
- **解决方案**：使用 Docker 自定义 bridge 网络，并配置服务别名。同时启用 DNS 缓存，将服务地址解析结果缓存至本地。

**问题2：ELK 日志收集占用资源过高**

- **现象**：Elasticsearch 和 Logstash 在开发服务器上运行时内存占用超过 4GB，导致其他服务运行缓慢。
- **原因分析**：ELK 组件默认 JVM 堆内存配置较高，不适合开发环境。
- **解决方案**：调整 Elasticsearch 和 Logstash 的 JVM 堆内存参数（各限制为 512MB），并设置日志采集频率和保留策略（保留 7 天）。

**问题3：分页查询深度分页性能问题**

- **现象**：标本列表查询翻到较后面的页码时，响应时间明显增加。
- **原因分析**：MySQL 的 LIMIT offset, size 在 offset 较大时需要扫描大量数据行。
- **解决方案**：采用游标分页（基于最后一条记录的 ID 查询）替代传统 OFFSET 分页，并配合 Elasticsearch 实现全文检索场景的优化分页。

**问题4：前端打包体积过大**

- **现象**：前端生产环境构建后的包体积达到 8.5MB，加载缓慢。
- **原因分析**：未进行有效的代码分割和依赖优化，Element Plus 全量引入。
- **解决方案**：配置 Vite 构建优化，Element Plus 改为按需引入，启用 Gzip 压缩，实现路由级代码分割。优化后打包体积降至 2.1MB。
