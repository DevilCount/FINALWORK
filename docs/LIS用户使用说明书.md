## 1. 系统概述

### 1.1 系统简介

本系统是一套基于微服务架构的实验室信息管理系统（Laboratory Information System，简称 LIS），面向医疗机构实验室的日常检验业务流程，提供从标本登记、签收、入库、检验到报告审核发布的全流程数字化管理能力。

系统参考卫宁健康 LIS 6.0 产品的功能设计理念，采用 Spring Cloud Alibaba 微服务技术栈和 Vue.js 前端框架构建，涵盖七大核心功能模块：

| 模块 | 功能说明 |
|------|---------|
| 用户管理 | 用户账号、角色权限、部门组织、菜单字典管理 |
| 标本管理 | 标本登记、签收、入库、状态流转、全流程追溯、条码管理 |
| 检验管理 | 检验申请、结果录入、报告审核发布、危急值管理、患者管理 |
| 设备管理 | 设备台账、联机通讯、校准记录、维护保养、状态监控 |
| HL7接口对接 | HIS系统对接、HL7消息解析、接口配置、消息监控 |
| 数据统计 | 工作量统计、标本统计、报告统计、设备统计、可视化图表 |
| AI辅助诊断 | 血常规/尿常规/肝功能智能分析、诊断规则引擎、医师审核 |

### 1.2 系统架构

系统后端采用微服务架构，按照业务领域划分为以下独立微服务：

| 微服务 | 端口 | 职责 |
|--------|------|------|
| lis-gateway | 8080 | API网关，路由转发、统一鉴权、限流熔断 |
| lis-auth | 8081 | 认证授权，JWT令牌管理 |
| lis-user | 8082 | 用户、角色、权限、部门、字典管理 |
| lis-specimen | 8083 | 标本全生命周期管理 |
| lis-report | 8084 | 检验申请、结果录入、报告管理 |
| lis-equipment | 8085 | 设备信息维护和联机通讯 |
| lis-hl7 | 8086 | HIS系统消息对接 |
| lis-statistics | 8087 | 业务数据统计与可视化 |
| lis-ai | 8088 | 检验结果智能分析 |

### 1.3 运行环境要求

| 项目 | 要求 |
|------|------|
| 浏览器 | Chrome 80+ / Firefox 75+ / Safari 13+ / Edge 80+ |
| 屏幕分辨率 | 1920×1080（推荐）、1366×768（最低） |
| 网络 | 需要访问服务器地址，建议100Mbps及以上带宽 |

## 2. 系统登录与认证

### 2.1 登录系统

1. 打开浏览器，输入系统访问地址（如 `http://localhost:5173`）
2. 进入登录页面，输入以下信息：
   - **用户名**：系统分配的账号（如 `admin`）
   - **密码**：对应的登录密码（初始密码为 `admin123`）
   - **验证码**：连续输入错误3次密码后，系统将自动启用图形验证码
3. 点击 **「登录」** 按钮

### 2.2 登录安全规则

| 规则 | 说明 |
|------|------|
| 密码错误锁定 | 连续输入密码错误5次，账号自动锁定30分钟 |
| 令牌有效期 | JWT访问令牌有效期为2小时，刷新令牌有效期为7天 |
| 自动续期 | 令牌过期后系统自动使用刷新令牌获取新令牌，无需重新登录 |
| 单点登录 | 同一账号不允许多处同时登录，后登录者自动挤退先登录者 |
| 会话超时 | 用户会话超时时间为30分钟，超时自动退出 |

### 2.3 修改密码

1. 登录系统后，点击右上角用户头像
2. 在下拉菜单中选择 **「修改密码」**
3. 输入旧密码和新密码（新密码需包含大小写字母和数字，8-20位）
4. 点击 **「确认」** 完成修改

### 2.4 退出登录

点击右上角用户头像，选择 **「退出登录」**，系统将清除登录状态并跳转回登录页面。

```
┌──────────────────────────────────────────────────────┐
│  顶部导航栏：系统Logo | 系统名称 | 消息通知 | 用户头像  │
├──────────┬───────────────────────────────────────────┤
│          │                                           │
│  左侧    │          右侧内容区                        │
│  菜单栏  │     ┌─────────────────────────┐           │
│          │     │   数据概览卡片区域        │           │
│  · 首页  │     │  (今日标本/报告/危急值等) │           │
│  · 标本  │     └─────────────────────────┘           │
│  · 报告  │     ┌──────────┬──────────────┐           │
│  · 设备  │     │ 趋势图表  │  分类统计     │           │
│  · AI    │     │ (折线图)  │  (饼图)      │           │
│  · 统计  │     └──────────┴──────────────┘           │
│  · 系统  │                                           │
│          │                                           │
└──────────┴───────────────────────────────────────────┘
```

### 3.2 仪表盘数据

仪表盘提供以下关键数据概览：

- **今日标本数**：当日登记的标本总量
- **待签收标本**：已登记但尚未签收的标本数量
- **检验中标本**：正在进行检验的标本数量
- **今日报告数**：当日生成的检验报告总量
- **待审核报告**：等待初审或终审的报告数量
- **危急值报警**：未处理的危急值数量（红色醒目提示）
- **设备在线率**：当前在线设备占比
- **趋势图表**：近7天/30天的标本和报告趋势

## 4. 标本管理

标本管理是实验室业务流程的核心模块，实现标本从登记到完成的全生命周期管理。

### 4.1 标本登记

**操作路径**：标本管理 → 标本登记

**操作步骤**：

1. 点击左侧菜单 **「标本管理」→「标本登记」**
2. 在标本登记页面，点击 **「新增」** 按钮
3. 填写标本登记信息：
   - **患者信息**：姓名、性别、年龄、联系电话、身份证号
   - **就诊信息**：科室、床号、临床诊断
   - **标本信息**：标本类型（血液/尿液/粪便/体液等）、检验项目（可多选）
   - **备注**：特殊情况说明
4. 点击 **「确认」** 提交登记

**登记成功后**：
- 系统自动生成标本编号（格式：标本类型代码+年月日+6位流水号）
- 自动生成条码，标本状态置为 **「已登记」**
- 可直接打印条码标签

### 4.2 标本签收

**操作路径**：标本管理 → 标本接收

**支持两种签收模式**：

#### 扫码签收
1. 在标本接收页面，使用条码扫描枪扫描标本条码
2. 系统自动识别并显示标本信息
3. 确认标本信息无误后，点击 **「签收」** 按钮
4. 标本状态更新为 **「已签收」**

#### 批量签收
1. 在标本接收页面，按条件筛选待签收标本（按送检科室、送检时间等）
2. 勾选需要签收的标本
3. 点击 **「批量签收」** 按钮
4. 系统批量更新标本状态

**签收注意事项**：
- 签收时系统自动检查标本是否在有效时间范围内
- 超时标本需注明原因方可签收
- 不合格标本可选择 **「拒收」**，填写拒收原因（溶血/脂血/量不足/容器错误/信息不符/其他）

### 4.3 标本入库

**操作路径**：标本管理 → 标本列表 → 入库操作

1. 在标本列表中筛选状态为 **「已签收」** 的标本
2. 选择需要入库的标本，点击 **「入库」**
3. 设置检验分组/岗位分配和优先级（普通/加急/紧急）
4. 确认入库，标本状态更新为 **「已入库」**

> 标本入库前必须已完成签收。入库后支持撤销操作，撤销后状态回退至「已签收」。

### 4.4 标本状态流转

标本在系统中严格按以下顺序流转：

```
已登记 → 已签收 → 已入库 → 检验中 → 已完成
  ↑                                          |
  └──────── 管理员回退（需记录原因）──────────┘
```

| 当前状态 | 目标状态 | 触发操作 | 操作角色 |
|---------|---------|---------|---------|
| — | 已登记 | 标本登记 | 护士 |
| 已登记 | 已签收 | 扫码/批量签收 | 检验师/接收员 |
| 已签收 | 已入库 | 排号入库 | 检验师 |
| 已入库 | 检验中 | 开始检验 | 检验师 |
| 检验中 | 已完成 | 结果录入完成 | 检验师 |
| 任意状态 | 前一状态 | 管理员回退 | 管理员 |

### 4.5 标本全流程追溯

**操作路径**：标本管理 → 标本追溯

1. 输入标本编号或条码进行查询
2. 系统展示完整的标本时间线，包括：
   - 标本基本信息（编号、类型、患者、检验项目）
   - 各状态节点的变更时间、操作人、操作内容
   - 关联的检验报告信息
3. 追溯数据按时间倒序排列，不可篡改

### 4.6 条码管理

- **自动生成**：标本登记时系统自动生成条码（Code 128 或 QR Code 格式）
- **标签打印**：支持单个和批量打印条码标签
- **条码补打**：支持补打操作，补打记录自动保存
- **标签内容**：标本编号、患者信息、标本类型、检验项目、条形码/二维码

### 4.7 不合格标本处理

1. 在签收环节或检验前识别不合格标本
2. 选择不合格原因：溶血、脂血、量不足、容器错误、信息不符、其他
3. 选择处理方式：
   - **退回重新采集**：系统自动向送检科室发送退回通知
   - **标注后继续检验**：记录不合格原因后继续流程
4. 不合格标本信息纳入统计模块，用于计算不合格率

## 5. 检验报告管理

检验报告管理覆盖检验申请、结果录入、报告审核发布、危急值管理等全链条。

### 5.1 检验申请

**操作路径**：报告管理 → 报告申请

**创建方式**：

#### 方式一：HIS系统自动获取（推荐）
- HIS系统通过HL7接口推送检验医嘱（ORM消息）
- 系统自动创建检验申请单，关联患者信息

#### 方式二：手工创建
1. 点击 **「新增申请」** 按钮
2. 填写申请信息：患者信息、检验项目、申请科室、申请医生、临床诊断、标本类型
3. 点击 **「确认」** 提交

**申请单状态**：待处理 → 已关联标本 → 检验中 → 已完成 / 已取消

> 超过24小时未关联标本的申请单自动标记为「超时预警」。

### 5.2 检验结果录入

**操作路径**：报告管理 → 结果录入

**录入方式**：

#### 手工录入
1. 在结果录入页面，选择待录入的标本/报告
2. 逐项录入检验结果：
   - 结果值
   - 结果单位
   - 参考范围（系统根据患者性别、年龄自动匹配）
   - 异常标记（系统自动判断，超出参考范围自动标记）
   - 备注
3. 点击 **「保存」** 提交结果

#### 仪器自动导入
- 检验设备通过串口/网络通讯自动传输检验结果
- 导入后需操作员确认

**结果录入规则**：
- 结果超出参考范围时，系统自动标记为异常（偏高↑/偏低↓），以醒目颜色显示
- 同一检验项目仅允许录入一次结果，修改需记录原因
- 结果录入完成后，标本状态自动更新为 **「已完成」**

### 5.3 报告审核

**操作路径**：报告管理 → 报告审核

系统采用 **两级审核机制**：

```
草稿 → 提交初审 → 待初审 → 初审通过 → 待终审 → 终审通过 → 已发布
                  ↓                      ↓
               初审退回              终审退回
                  ↓                      ↓
                草稿                  待初审
```

#### 初审（检验医师）
1. 在待初审列表中选择报告
2. 查看检验结果和患者信息
3. 审核操作：
   - **通过**：报告进入终审队列
   - **退回**：填写退回原因，报告状态回退至草稿

#### 终审（审核医师/主管）
1. 在待终审列表中选择报告
2. 进行终审操作：
   - **通过**：报告状态更新为 **「已发布」**，临床端可查看
   - **退回**：填写退回原因，报告状态回退至待初审

> 同一审核人不可对同一报告进行初审和终审（需不同人员操作）。

### 5.4 报告发布与回收

#### 发布
- 报告终审通过后自动发布
- 发布后临床医生端可查看报告内容
- 系统通过HL7接口自动将结果回传至HIS系统

#### 回收
1. 在已发布报告列表中选择需要回收的报告
2. 填写回收原因
3. 确认回收，临床端不可再查看
4. 回收后的报告需重新走审核流程

### 5.5 报告打印

**操作路径**：报告管理 → 报告查询 → 打印

- 仅 **已发布** 状态的报告支持打印
- 支持 **单个打印** 和 **批量打印**
- 打印模板支持自定义配置（格式、Logo、页眉页脚）
- 每次打印操作记录打印时间、打印人和份数
- 补打报告自动加盖 **「补打」** 标识

### 5.6 报告查询

**操作路径**：报告管理 → 报告查询

**查询条件**（支持组合查询）：
- 患者姓名/ID、住院号/门诊号
- 报告编号、标本编号
- 检验项目、报告状态
- 时间范围、科室、审核医师

**查询结果**：
- 报告列表（分页展示）
- 报告详情（含患者信息、标本信息、检验结果、参考范围、审核信息）
- 异常指标醒目标识
- 支持导出为Excel格式

### 5.7 危急值管理

**操作路径**：报告管理 → 危急值管理

#### 危急值自动识别
- 系统预置常见检验项目的危急值阈值
- 检验结果录入时，系统自动判断是否触发危急值
- 触发危急值时，系统立即发送报警通知（弹窗+消息推送）

#### 危急值处理流程

```
待处理 → 已通知 → 已确认 → 已处理
  ↓                              ↑
  └──── 直接处理 ────────────────┘
```

1. **通知**：系统自动向检验医师和临床医生发送报警通知
2. **确认**：临床医生确认收到通知
3. **处理**：检验医师记录处理措施、通知时间、通知对象、临床反馈
4. **超时升级**：危急值必须在30分钟内处理，超时未处理系统自动升级报警

> 危急值处理记录不可删除，仅支持追加处理信息。

## 6. 设备管理

### 6.1 设备台账管理

**操作路径**：设备管理 → 设备列表

**设备信息字段**：

| 字段 | 说明 |
|------|------|
| 设备编号 | 全局唯一，系统自动生成或手工录入 |
| 设备名称 | 设备的名称 |
| 设备型号 | 设备型号规格 |
| 生产厂商 | 设备制造商 |
| 出厂日期 / 购置日期 | 设备时间信息 |
| 安装位置 | 设备所在实验室位置 |
| 设备状态 | 正常 / 维修中 / 已停用 / 已报废 |
| 负责人 / 联系电话 | 设备责任人信息 |

**设备操作**：
- **新增设备**：填写设备信息，点击确认
- **编辑设备**：修改设备信息
- **删除设备**：逻辑删除，保留历史数据
- **状态变更**：记录变更原因和时间

### 6.2 设备状态监控

**操作路径**：设备管理 → 设备监控

实时监控设备运行状态：
- **在线/离线状态**：设备通讯连接状态
- **今日检验量**：设备当日完成的检验数量
- **累计检验量**：设备历史总检验数量
- **通讯状态**：最近一次通讯时间
- **异常计数**：设备异常/错误次数

### 6.3 设备校准记录

**操作路径**：设备管理 → 设备校准

1. 选择设备，新增校准记录
2. 填写校准信息：
   - 校准类型：日常校准 / 定期校准 / 维修后校准
   - 校准日期、校准人、校准机构
   - 校准结果：合格 / 不合格
   - 下次校准日期
3. 保存校准记录

**校准规则**：
- 校准结果为 **不合格** 时，设备状态自动变更为 **维修中**，并通知负责人
- 系统根据校准周期自动计算下次校准日期，提前7天发送校准到期提醒
- 校准记录不可删除，仅支持追加

### 6.4 设备维护保养

**操作路径**：设备管理 → 设备维护

1. 选择设备，新增维护记录
2. 填写维护信息：
   - 维护类型：日常维护 / 定期保养 / 故障维修
   - 维护内容、维护人
   - 维护结果
   - 耗材使用情况
   - 故障维修时需记录：故障现象、故障原因、维修措施
3. 保存维护记录

**维护规则**：
- 系统支持维护计划的制定和到期提醒
- 维护记录纳入设备使用统计分析

## 7. AI辅助诊断

AI辅助诊断模块基于检验结果数据，提供智能化的结果分析和辅助诊断建议。

### 7.1 发起AI分析

**操作路径**：AI诊断 → 智能分析

1. 在智能分析页面，选择待分析的检验报告
2. 选择诊断类型：
   - **血常规分析**：分析白细胞、红细胞、血红蛋白、血小板等指标
   - **尿常规分析**：分析尿比重、pH值、尿蛋白、葡萄糖等指标
   - **肝功能分析**：分析ALT、AST、胆红素、白蛋白等指标
3. 点击 **「开始分析」** 按钮
4. 系统基于规则引擎进行分析，生成诊断结果

### 7.2 分析结果查看

分析结果包含以下内容：

| 内容 | 说明 |
|------|------|
| 各指标分析结果 | 正常/偏高/偏低，异常指标高亮标注 |
| 指标组合分析 | 多项指标联合异常的综合分析结论 |
| 临床意义解释 | 异常指标可能的临床意义 |
| 初步诊断建议 | 按可能性排序的诊断建议列表 |
| 置信度标注 | 高（≥80%，绿色）/ 中（60%-80%，黄色）/ 低（<60%，橙色） |
| 依据说明 | 每条建议的异常指标和推理逻辑 |

> AI分析结果明确标注「仅供参考，不作为临床诊断依据」。

### 7.3 医师审核

**操作路径**：AI诊断 → 诊断记录

1. 在诊断记录列表中选择待审核的AI分析结果
2. 审核操作：
   - **采纳**：AI建议关联到检验报告
   - **部分采纳**：选择性采纳部分建议
   - **否决**：否决AI建议，需填写否决原因（反馈数据可用于模型优化）
3. 审核记录包含审核人、审核时间、审核结果和审核意见
4. AI建议在报告中以 **「AI辅助分析」** 标识独立展示

### 7.4 诊断规则管理

**操作路径**：AI诊断 → 诊断规则

管理员可管理AI诊断规则引擎：

- **查看规则列表**：按分类查看所有诊断规则
- **新增规则**：定义规则名称、分类、条件、动作、优先级、置信度阈值
- **编辑规则**：修改规则参数
- **启用/禁用规则**：控制规则是否参与分析
- **测试规则**：使用测试数据验证规则效果
- **规则分类**：按血常规、尿常规、肝功能等分类管理

## 10. HL7接口与HIS对接

### 10.1 接口配置

**操作路径**：接口管理 → 接口配置

配置与HIS系统的对接参数：
- 接口类型：MLLP/HTTP
- 连接地址和端口
- 消息编码格式
- 启用/禁用接口

### 10.2 消息监控

**操作路径**：接口管理 → 消息监控

实时监控HL7消息的收发情况：
- 消息列表（消息类型、方向、状态、时间）
- 消息详情（原始消息内容、解析结果）
- 异常消息处理
- 消息统计

### 10.3 数据对接流程

#### HIS → LIS（接收数据）
1. **患者信息同步**：接收HIS推送的患者基本信息（ORM消息PID段）
2. **检验医嘱下载**：接收HIS推送的检验医嘱（ORM消息ORC/OBR段），自动创建检验申请单

#### LIS → HIS（回传数据）
1. **检验结果回传**：报告发布后，系统自动将检验结果组装为ORU消息回传至HIS
2. **消息确认应答**：所有消息收发均返回ACK确认消息

#### 异常处理
- 消息发送失败自动重试（最多3次，间隔1分钟、5分钟、30分钟）
- 超过重试次数转入异常队列，等待人工处理
- 消息根据控制ID（MSH-10）进行去重
- HIS不可用时消息进入等待队列，恢复后自动重发

## 13. 常见问题与故障排除

### 13.1 登录相关

| 问题 | 解决方案 |
|------|---------|
| 忘记密码 | 联系系统管理员重置密码 |
| 账号被锁定 | 等待30分钟自动解锁，或联系管理员手动解锁 |
| 验证码不显示 | 刷新页面或检查浏览器是否禁用了图片加载 |
| 登录后页面空白 | 清除浏览器缓存后重试，或更换Chrome浏览器 |

### 13.2 标本相关

| 问题 | 解决方案 |
|------|---------|
| 条码无法扫描 | 检查条码是否污损，尝试手工输入标本编号查询 |
| 标本签收超时提示 | 确认标本采集时间，填写超时原因后继续签收 |
| 不合格标本退回 | 选择不合格原因，系统自动通知送检科室重新采集 |
| 标本状态异常 | 联系管理员进行状态回退操作 |

### 13.3 报告相关

| 问题 | 解决方案 |
|------|---------|
| 报告审核退回 | 查看退回原因，修改后重新提交审核 |
| 危急值报警未收到 | 检查消息通知设置，确认网络连接正常 |
| 报告打印格式异常 | 检查打印模板配置，确认打印机设置 |
| 报告无法发布 | 确认报告已完成终审通过 |

### 13.4 设备相关

| 问题 | 解决方案 |
|------|---------|
| 设备通讯中断 | 检查网络/串口连接，重启通讯服务 |
| 设备状态异常 | 联系设备负责人，进行维护或校准 |
| 校准到期提醒 | 及时安排设备校准，更新校准记录 |

### 13.5 系统相关

| 问题 | 解决方案 |
|------|---------|
| 页面加载缓慢 | 检查网络连接，清除浏览器缓存 |
| 菜单显示不全 | 确认用户角色权限配置是否正确 |
| 数据导出失败 | 检查数据量是否过大，尝试缩小查询范围 |
| 接口调用失败 | 检查服务是否正常运行，查看网关日志 |
