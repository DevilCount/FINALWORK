## 2. 环境准备

### 2.1 安装JDK 17

#### CentOS / RHEL

```bash
# 安装OpenJDK 17
sudo yum install -y java-17-openjdk java-17-openjdk-devel

# 验证安装
java -version
```

#### Ubuntu / Debian

```bash
# 安装OpenJDK 17
sudo apt update
sudo apt install -y openjdk-17-jdk

# 验证安装
java -version
```

#### 配置环境变量

```bash
# 编辑配置文件
sudo vim /etc/profile

# 添加以下内容
export JAVA_HOME=/usr/lib/jvm/java-17-openjdk
export PATH=$JAVA_HOME/bin:$PATH

# 使配置生效
source /etc/profile
```

### 2.2 安装Maven

```bash
# 下载Maven
wget https://dlcdn.apache.org/maven/maven-3/3.9.6/binaries/apache-maven-3.9.6-bin.tar.gz

# 解压
tar -xzf apache-maven-3.9.6-bin.tar.gz
sudo mv apache-maven-3.9.6 /opt/maven

# 配置环境变量
sudo vim /etc/profile

# 添加以下内容
export MAVEN_HOME=/opt/maven
export PATH=$MAVEN_HOME/bin:$PATH

# 使配置生效
source /etc/profile

# 验证安装
mvn -version
```

#### 配置Maven阿里云镜像

编辑 `/opt/maven/conf/settings.xml`：

```xml
<mirrors>
    <mirror>
        <id>aliyun</id>
        <name>Aliyun Maven Mirror</name>
        <url>https://maven.aliyun.com/repository/public</url>
        <mirrorOf>central</mirrorOf>
    </mirror>
</mirrors>
```

### 2.3 安装MySQL

#### CentOS / RHEL

```bash
# 添加MySQL官方仓库
sudo rpm -Uvh https://dev.mysql.com/get/mysql80-community-release-el7-7.noarch.rpm

# 安装MySQL
sudo yum install -y mysql-community-server

# 启动MySQL服务
sudo systemctl start mysqld
sudo systemctl enable mysqld

# 获取临时密码
sudo grep 'temporary password' /var/log/mysqld.log

# 安全配置
mysql_secure_installation
```

#### Ubuntu / Debian

```bash
# 安装MySQL
sudo apt update
sudo apt install -y mysql-server

# 启动MySQL服务
sudo systemctl start mysql
sudo systemctl enable mysql

# 安全配置
sudo mysql_secure_installation
```

#### 创建应用账号

```sql
-- 登录MySQL
mysql -u root -p

-- 创建用户
CREATE USER 'lis'@'%' IDENTIFIED BY 'YourStrongPassword123!';

-- 授权
GRANT ALL PRIVILEGES ON lis_*.* TO 'lis'@'%';
FLUSH PRIVILEGES;
```

### 2.4 安装Redis

#### CentOS / RHEL

```bash
# 安装EPEL仓库
sudo yum install -y epel-release

# 安装Redis
sudo yum install -y redis

# 启动Redis服务
sudo systemctl start redis
sudo systemctl enable redis

# 验证
redis-cli ping
```

#### Ubuntu / Debian

```bash
# 安装Redis
sudo apt update
sudo apt install -y redis-server

# 启动Redis服务
sudo systemctl start redis-server
sudo systemctl enable redis-server

# 验证
redis-cli ping
```

#### 配置Redis

编辑 `/etc/redis/redis.conf`（Ubuntu）或 `/etc/redis.conf`（CentOS）：

```conf
# 绑定地址（生产环境建议绑定内网IP）
bind 127.0.0.1

# 设置密码
requirepass YourRedisPassword123!

# 持久化配置
appendonly yes
```

重启Redis：

```bash
sudo systemctl restart redis
```

### 2.5 安装Nacos（可选）

如果使用Nacos作为服务注册与配置中心：

```bash
# 下载Nacos
wget https://github.com/alibaba/nacos/releases/download/2.3.0/nacos-server-2.3.0.tar.gz

# 解压
tar -xzf nacos-server-2.3.0.tar.gz
cd nacos/bin

# 单机模式启动
sh startup.sh -m standalone

# 访问控制台
# http://localhost:8848/nacos
# 默认账号：nacos / nacos
```

### 2.6 安装Node.js

```bash
# 使用nvm安装Node.js
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.bashrc

# 安装Node.js 18 LTS
nvm install 18
nvm use 18

# 验证安装
node -v
npm -v

# 配置npm淘宝镜像
npm config set registry https://registry.npmmirror.com
```

## 3. 数据库配置

### 3.1 创建数据库

系统采用分库策略，每个微服务拥有独立的数据库：

```sql
-- 创建数据库
CREATE DATABASE lis_user_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_specimen_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_report_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_equipment_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_hl7_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_statistics_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE lis_ai_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### 3.2 执行初始化脚本

按顺序执行SQL脚本：

```bash
cd lis-backend/sql

# 执行建表脚本
mysql -u lis -p lis_user_db < 01_lis_user.sql
mysql -u lis -p lis_specimen_db < 02_lis_specimen.sql
mysql -u lis -p lis_report_db < 03_lis_report.sql
mysql -u lis -p lis_equipment_db < 04_lis_equipment.sql
mysql -u lis -p lis_hl7_db < 05_lis_hl7.sql
mysql -u lis -p lis_statistics_db < 06_lis_statistics.sql
mysql -u lis -p lis_ai_db < 07_lis_ai.sql

# 执行初始化数据脚本
mysql -u lis -p lis_user_db < 08_init_data.sql
```

## 6. Docker部署

### 6.1 镜像构建

#### 6.1.1 后端Dockerfile

在各微服务目录下创建 `Dockerfile`：

```dockerfile
FROM openjdk:17-jdk-slim

WORKDIR /app

COPY target/*.jar app.jar

EXPOSE 8080

ENTRYPOINT ["java", "-jar", "app.jar"]
```

#### 6.1.2 前端Dockerfile

在 `lis-web` 目录下创建 `Dockerfile`：

```dockerfile
FROM node:18-alpine AS builder

WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine

COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

#### 6.1.3 构建镜像

```bash
# 构建后端镜像
cd lis-backend
docker build -t lis-gateway:1.0.0 -f lis-gateway/Dockerfile lis-gateway
docker build -t lis-auth:1.0.0 -f lis-auth/Dockerfile lis-auth
# ... 其他服务类似

# 构建前端镜像
cd lis-web
docker build -t lis-web:1.0.0 .
```

### 6.2 容器编排

#### 6.2.1 docker-compose.yml

```yaml
version: '3.8'

services:

      - mysql

      - SPRING_DATASOURCE_URL=jdbc:mysql://mysql:3306/lis_user_db

      - lis-network
```

## 7. 生产环境部署

### 7.1 Nginx配置

#### 7.1.1 前端静态资源部署

将前端构建产物部署到Nginx：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/lis-web/dist;
    index index.html;

    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API代理
    location /api/ {
        proxy_pass http://localhost:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1000;
}
```

#### 7.1.2 HTTPS配置（推荐）

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/nginx/ssl/your-domain.com.crt;
    ssl_certificate_key /etc/nginx/ssl/your-domain.com.key;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    # ... 其他配置同上
}

# HTTP重定向到HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}
```

### 7.2 服务监控

#### 7.2.1 Spring Boot Actuator

各微服务已集成Actuator，可访问以下端点：

| 端点 | 说明 |
|------|------|
| /actuator/health | 健康检查 |
| /actuator/info | 应用信息 |
| /actuator/metrics | 指标数据 |
| /actuator/prometheus | Prometheus格式指标 |

#### 7.2.2 健康检查脚本

```bash
#!/bin/bash

services=("gateway:8080" "auth:8081" "user:8082" "specimen:8083" "report:8084" "equipment:8085" "hl7:8086" "statistics:8087" "ai:8088")

for service in "${services[@]}"; do
    name=$(echo $service | cut -d: -f1)
    port=$(echo $service | cut -d: -f2)
    
    response=$(curl -s -o /dev/null -w "%{http_code}" http://localhost:$port/actuator/health)
    
    if [ "$response" = "200" ]; then
        echo "✓ $name is healthy"
    else
        echo "✗ $name is unhealthy (HTTP $response)"
    fi
done
```

## 8. 常见问题

### Q1: 服务启动失败，提示数据库连接错误

**解决方案**：
1. 检查MySQL服务是否启动
2. 检查数据库连接配置是否正确
3. 检查数据库用户权限
4. 检查防火墙是否开放3306端口

```bash
# 检查MySQL服务状态
systemctl status mysql

# 测试数据库连接
mysql -u lis -p -h localhost lis_user_db
```

### Q2: Redis连接失败

**解决方案**：
1. 检查Redis服务是否启动
2. 检查Redis密码配置
3. 检查防火墙是否开放6379端口

```bash
# 检查Redis服务状态
systemctl status redis

# 测试Redis连接
redis-cli -a YourRedisPassword123! ping
```

### Q3: 前端无法访问后端API

**解决方案**：
1. 检查后端服务是否启动
2. 检查API网关配置
3. 检查跨域配置
4. 检查浏览器控制台错误信息

### Q4: JWT Token验证失败

**解决方案**：
1. 检查JWT密钥配置是否一致
2. 检查Token是否过期
3. 检查Redis中Token是否存在

### Q5: 服务端口被占用

**解决方案**：

```bash
# 查看端口占用
netstat -tlnp | grep :8080

# 终止占用进程
kill -9 <PID>
```
